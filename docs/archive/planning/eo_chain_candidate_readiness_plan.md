# EO Candidate Chain Readiness Plan

Status: implementation foundation.

## Purpose

This block turns the employer-origin candidate queue into a portfolio-level
readiness report. It keeps the existing single-candidate chain driver intact,
but adds a read-only JSON/Markdown report that answers:

- which employer-origin candidates can safely move to the next bounded step,
- which candidates are monitor-only,
- which candidates are stopped behind manual review or explicit approval,
- which Safety Zone the next step would enter,
- which single command is the first operator-friendly follow-up.

The report is intended to sit between DETAIL evidence discovery and any
apply-controlled EO/connector work. It is not a mutation agent.

## Boundary

The candidate-chain readiness report is read-only:

- no candidate URL write,
- no gate review write,
- no evidence write,
- no connector artifact write,
- no connector registration,
- no source activation,
- no scheduler change.

Commands in the report are planned commands only. They must still be run
explicitly by the operator and must respect dry-run/apply boundaries of the
called agent.

## Why this is the next block after DETAIL-005

DETAIL-005 clarified the detail-evidence report contract by separating
preliminary detail candidates from authoritative assessments and supported
details. The next bottleneck is no longer report wording; it is deciding which
EO candidates should proceed through the chain and which should remain stopped.

The readiness report therefore consolidates the current candidate state into one
reviewable queue before any further apply-controlled action is taken.

## Inputs

The script reads:

- `employer_origin_source_candidates`, including `candidate_url`,
- `employer_origin_candidate_gate_reviews`,
- existing connector artifact files on disk,
- existing recheck policy from `aggregator_discovery_policy`,
- existing single-candidate chain decision logic from
  `run_employer_origin_agent_chain`.

## Outputs

When called with `--write-report`, the script writes:

- JSON report,
- Markdown report,
- summary counts by next action, candidate status, and Safety Zone,
- first actionable command for operator convenience.

Default output directory:

    exports/eo_chain_candidate_readiness/

Recommended command:

    python -m scripts.run_employer_origin_candidate_queue_agent \
      --allow-repair \
      --write-report \
      --print-next-command \
      --benchmark-label eo_chain_candidate_readiness_after_detail005 \
      --reviewed-by jens

## Interpretation rules

- `queue_items` are ordered recommendations, not an apply list.
- `command` means planned command only; the readiness report never executes it.
- `manual_review_or_stop_count` marks candidates that should not be progressed
  automatically from this report.
- `first_actionable_command` is a convenience pointer, not automatic execution.

## System impact

Discovery: no new discovery or external provider call.

Evidence: no new evidence write; the report consumes current gate/evidence
state only.

Candidate/Gate: improves operator visibility for candidate chain progression;
no gate mutation.

Connector: no connector files are generated by this report; it can recommend
artifact-generation planning commands only.

Bronze/Silver/Gold: no job-layer write.

UI/Observability: produces structured export data that can later feed STATE or
Operations views without turning Tavily or detail recovery into a new sensor.
