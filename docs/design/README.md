# Design and Platform Identity

This directory defines the visual language used for project-facing diagrams, dashboard mockups, presentation assets and future frontend work.

The goal is not decorative styling. Visual design is treated as part of the platform architecture because it communicates data flow, source quality, risk boundaries and engineering maturity.

The project style is called **Deep Ocean Intelligence**.

## Documents

- [Visual Identity](visual_identity.md) defines colors, language strategy, visual principles and project terminology for diagrams and dashboards.
- [Documentation Design Rules](documentation_design_rules.md) defines how documentation updates should preserve this visual identity over time.

## Scope

These rules apply to:

- README hero graphics and project overview visuals
- architecture diagrams
- data-flow diagrams
- source-landscape and source-risk visuals
- dashboard mockups
- presentation decks
- future frontend components

They do not require every text-only documentation change to create or update a graphic. They do require that any touched visual material remains consistent with the project identity.
